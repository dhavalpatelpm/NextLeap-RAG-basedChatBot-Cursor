# Phase 1 Data Layout

- **reference/** — Curated JSON per cohort (and `platform.json`). Includes deep structure: curriculum by week, program timeline, instructors, tools, schedule, alumni support, salary, reviews. Edit to add or correct content; merge uses them as the base.
- **scraped/** — Output of the Playwright scraper (one JSON per cohort). Regenerated by `python scripts/run_phase1.py`. Contains live data: prices, cohort start dates, price increase, instructors, schedule, salary, curriculum/tools when extracted from the page.
- **canonical/** — Merged reference + scraped (one JSON per cohort). Input for Phase 2 (RAG). Regenerated by Phase 1 merge. Holds all fields needed for the chatbot (instructors, average/max salary, reviews, cohort start, price increase, live hours, weekly commitment, projects count, curriculum by week, program timeline, tools learned, alumni support).
- **course_details/** — One folder per course (`product_management/`, `ux_design/`, etc.) each with `course.json`. Used so the chatbot can load only the relevant course when answering. **Always return the course’s `source_url`** (in `course_details/index.json`) with every answer. Data here is a copy of the updated reference; re-run sync when reference or canonical changes (see below).

## Regenerating data

From repo root:

```bash
# Install deps and Playwright browsers (first time only)
pip install -r requirements.txt
playwright install chromium

# Scrape all course pages and merge to canonical
python scripts/run_phase1.py

# Merge only (no scrape), e.g. after editing reference/
python scripts/run_phase1.py --no-scrape

# Scrape only
python scripts/run_phase1.py --scrape-only

# Single cohort
python scripts/run_phase1.py --cohorts product_management
```

## Syncing course_details from reference

After updating `data/reference/*.json`, refresh the per-course folder so answers use the latest data:

```bash
cp data/reference/product_management.json data/course_details/product_management/course.json
cp data/reference/ux_design.json data/course_details/ux_design/course.json
cp data/reference/data_analytics.json data/course_details/data_analytics/course.json
cp data/reference/business_analytics.json data/course_details/business_analytics/course.json
cp data/reference/genai_bootcamp.json data/course_details/genai_bootcamp/course.json
```

Or run `python scripts/sync_course_details.py` if that script exists.

## Phase 2 — RAG vector store

- **vector_store/** — Chroma persistence directory. Created when you run Phase 2. Contains embeddings of course chunks (by section: overview, fee, instructors, curriculum, schedule, career, tools, etc.) for retrieval.

Build the RAG index (after course_details is populated):

```bash
python scripts/run_phase2.py
```

Retrieval uses `data/course_details/` as the source of course content; each chunk carries `source_url` so Phase 3 can return the correct course page URL with answers.

## Phase 3 — Chatbot (Groq)

The chatbot uses **Groq** as the LLM. Set your API key in a `.env` file at the repo root (see `.env.example`):

- `GROQ_API_KEY` — required
- `GROQ_MODEL` — optional, default `llama-3.1-8b-instant`

Run a quick test (after building the Phase 2 index):

```bash
python scripts/run_phase3_chat.py "What is the fee for Product Manager Fellowship?"
```
